# 府城拾光 ｜ 台南鐵路地下化空間房價熱力地圖與都市計畫分析系統
> 透過細緻的襯線文字與柔和的大地色調，將城市「縫合」的溫度，具體呈現於數據探勘的輿圖之上。

本專案是一個兼具「高雅文青美學」與「數據科學分析」的都市計畫房價空間輿圖系統。以台南鐵路地下化綠園道計畫長廊為研析核心，透過 1.5 公里的空間緩衝區 (Buffer Zone) 與多維度熱力漸層，呈現都市地景重建與住宅市場價位之動態變遷。

---

## 🎨 文青質感美學設計概念
* **色調 pairing**: 採用細膩柔和的大地色系。底色使用純淨高雅的溫暖亞麻白 (`#fcfbf7`、`#faf7f0`)，點綴充滿泥土氣息與歷史古韻的磚瓦釉紅 (Terracotta / 釉陶紅 `#c35a3e`) 為主調、輔以寧靜草木綠 (`#5d614e`)、石墨灰與紙質紋理，呈現典雅雋永、如紙本裝幀的視覺韻律。
* **字體 pairing**: 標題與空間人文詮釋採用「Noto Serif TC (源石繁體襯線體)」與「Playfair Display」，呈現復古老派與現代簡約極致碰撞；UI 數據指標採用極簡的高辨識度「Inter」與「JetBrains Mono」等寬字體，確保高精度數據呈現時的明晰與冷靜。
* **誠實無偽的無干擾介面**: 拒絕低質感、科技扮演的雜亂指示器，聚焦於簡潔有力的卡片佈局與地圖載體，回歸「城市再造、微光生活」的空間命題。

## 🌟 核心功能一覽

1. **空間中軸與緩衝範圍 (Turf.js 整合)**:
   * 以紅磚虛線清晰標示「林森路 → 生產路 → 大同路」台南鐵轨地下長廊中軸計畫。
   * 通過 **Turf.js** 運算，動態繪製 **500公尺、1000公尺、1500公尺** 的半透明琥珀色覆蓋分析範圍（緩衝區），完美呈現站點輻射力。
2. **多維度篩選分析儀**:
   * **時間**、**建物型態**、**單價區間/坪**、**緩衝半徑** 進行全交互式無縫過濾。
   * 同步呈現每坪平均價、中位數、最高/最低行情。
3. **鐵路空間衰減分析 (Distance Decay Curves)**:
   * 藉由地理座標計算，將交易樣分布為 `0-250m`、`250-500m`、`500-1000m`、`1000m-1500m` 之距離帶，以雅致的柱狀圖表解構「距離捷運綠園道越近，溢價越高」的微觀經濟學模型。
4. **AI 空間發展計畫學家 (Gemini API 深度全棧整合)**:
   * 由 **Gemini 3.5 Flash** 擔綱城市研究主筆，根據當前地圖過濾數據，撰寫極具文人情感、富有溫度的台南都市更新與縫合展望剖析，解答您的土地論述。
5. **雙軌實價登錄 CSV 整合上傳**:
   * **開箱即用**: 提供 `process-data.js` 生成 150 筆符合台南真實重劃行情（東區、永康、中西區、南區）的地段實務樣本。
   * **自主匯入**: 支援直接上傳或拖入內政部地政司「不動產交易實價登錄」原始 CSV。

## 📊 如何下載與放入內政部實價登錄資料

為了精準呈現真實行情，您可直接至政府公開門戶下載最新整併數據：

1. **前往政府資料開放平台**: 
   * 搜尋 `本期發布之不動產實價登錄批次資料` (內政部地政司發布)。
   * 或搜尋 `實價登錄`。
2. **尋找台南市數據**:
   * 下載整批壓縮檔 (.zip) 或主 CSV 檔案。
   * 解壓縮後，找到代表 **台南市買賣案件實價登錄** 的原始 CSV (`lvr_land_a.csv`)。
3. **匯入系統**:
   * **方式一 (推薦)**: 啟動本程式後，將該檔案直接「拖放」或「點選」上傳至專案左下角的 **「自訂實價登錄 CSV 數據匯入」** 區塊。後端會自動完成地段比對、Nominatim 經緯解析、Turf 空間距離檢索、清洗溢出值、並將全新數據併入系統地圖重載。
   * **方式二**: 將該檔案更名為 `greenway_house_geocoded.csv`（確保具有 lat, lng 或土地位置建物門牌，總價等欄位）並替換 `/data/` 底下之檔案。

## 🚀 專案啟動與運行指南

本專案採用 **Express.ts + Vite + React 18** 之高性能 full-stack 框架，API Key 有安全代理，全過程流暢：

### 1. 安裝環境與依賴
我們已在 workspace 為您備妥主要依賴，如需手動部署，請執行：
```bash
npm install
```

### 2. 環境變數配置 (.env)
系統自動讀取您在 Google AI Studio Secrets 中存放的 `GEMINI_API_KEY`；若在本地運行，請於根目錄將 `.env.example` 命名為 `.env` 並填入金鑰：
```env
GEMINI_API_KEY="您的_Gemini_API_Key"
```

### 3. 開發模式啟動
執行以下指令，系統會同時啟動 Node.js/TypeScript 服務器整合 Vite 的熱加載中間件，並自動綁定至 `3000` 埠口：
```bash
npm run dev
```
啟動後，開啟瀏覽器造訪：`http://localhost:3000`

### 4. 編譯與生產模式
一鍵 compile 所有前後端 TypeScript 代碼並 bundling 為單一輕量的 CJS server，極速部署：
```bash
# 編譯打包
npm run build

# 生產環境啟動
npm run start
```

## 📁 檔案樹說明
* `/server.ts` : 核心 Express 後端程式。統合 `/api/properties` 的數據讀取與手動上傳清洗；配置 `@google/genai` 整合。
* `/process-data.js` : 地理位置計算主處理腳本。解析 Turf.js 中軸模型，防範 Nominatim 頻率限制之逆境演算法，生成精選樣本備用。
* `/src/App.tsx` : 文青質雅風地標控制台、空間衰減柱狀分析儀、AI 交互面板、Leaflet 動態地圖圖層管理。
* `/src/index.css` : 主題面板核心字型 `@theme` 配色架構定義。
